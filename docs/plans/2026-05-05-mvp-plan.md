# AI Memory MVP 实施方案

> **目标**: 最小闭环验证 — 用户能导入ChatGPT/Claude对话 → 在Claude/Cursor中通过MCP搜索记忆 → 验证1000用户付费意愿
>
> **原则**: 刘小排框架 — 只保留核心功能，一周内上线，花1000元找1000用户，10%付费率回本

---

## 现状盘点

| 组件 | 状态 | 差距 |
|------|------|------|
| Web App (Next.js) | ✅ 运行中，有上传/搜索/浏览/统计 | 没有用户系统，对话数据是session隔离 |
| MCP Server | ✅ 6个工具（save/search/get/delete/stats/export） | 仅stdio模式，未发布pip包，无HTTP transport |
| Chrome Extension | ⚠️ 有content scripts框架 | 未完成，未上架Chrome Web Store |
| 存储层 | ✅ SQLite + FTS5 | 够用 |
| 定价/支付 | ❌ 无 | MVP不需要 |

---

## MVP 核心闭环（最小可行产品）

```
用户旅程:
1. 访问 aimemory.pro → 上传 ChatGPT/Claude 导出文件
2. 在 Web 上浏览、搜索自己的对话历史
3. 安装 MCP Server (pip install aimemory-mcp)
4. 在 Claude Desktop / Cursor 中直接搜索自己的记忆
5. (可选) 付费解锁云端同步 + AI分析
```

**砍掉的**: Chrome Extension（MVP不做）、团队功能、E2E加密、知识图谱、记忆注入

## 实施任务（7天计划）

### Day 1-2: MCP Server 发布

#### Task 1: 添加 HTTP Transport 模式
**文件**: `mcp-server/aimemory_mcp/server.py`

在现有 stdio 基础上加 HTTP/SSE transport，让用户可以通过远程访问：

```python
# 在 server.py 底部修改
if __name__ == "__main__":
    import sys
    transport = os.environ.get("AIMEMORY_TRANSPORT", "stdio")
    if transport == "http":
        mcp.run(transport="sse", host="0.0.0.0", port=int(os.environ.get("AIMEMORY_PORT", "8090")))
    else:
        mcp.run()
```

## 验证标准（刘小排框架）

| 指标 | 目标 | 放弃线 |
|------|------|--------|

## MVP 不做的事情（明确排除）

- ❌ Chrome Extension（复杂度高，MCP优先）
- ❌ 用户注册/登录系统（session够用）
- ❌ 支付系统（先收集邮箱，验证需求后再加）
- ❌ 云端同步（Pro功能，验证后加）
- ❌ AI分析/知识图谱（Pro功能）
- ❌ 记忆注入（需要Chrome Extension）
- ❌ 移动端适配（桌面优先）
- ❌ 国际化（英文优先，中文后续）
